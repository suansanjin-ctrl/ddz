# 云上斗地主

一个免构建的纯前端斗地主网页，支持 3 人联机。

联机方式不是自建后端，而是：

- 浏览器内用 WebRTC DataChannel 直连
- 房主负责发牌、规则校验和状态同步
- 通过手动复制 `Offer / Answer` 完成配对

## 本地启动

推荐不要直接双击 `index.html`，而是起一个本地静态服务：

```bash
cd /Users/dzh/Desktop/斗地主
python3 -m http.server 4173
```

然后打开：

- [http://localhost:4173/index.html](http://localhost:4173/index.html)

## 联机流程

1. 三个人分别打开页面。
2. 其中一人点击“成为房主”。
3. 房主分别为 `2 号位`、`3 号位` 点击“生成 Offer”。
4. 另外两人把对应 Offer 粘贴到“加入房间”区域，选好座位，点击“生成 Answer 并加入”。
5. 两位加入者把自己的 Answer 发回房主。
6. 房主把两个 Answer 分别粘贴回对应座位，点击“接入 2 号位 / 接入 3 号位”。
7. 两个座位都显示“已连接”后，房主点击“开始新一局”。

## 限制

- 这是纯前端方案，所以没有中心服务器，也没有自动房间匹配。
- 首次联机需要手动交换 SDP 文本，体验上偏工程化，但部署最简单。
- WebRTC 连通性依赖浏览器网络环境；在部分严格 NAT 环境下，只有 STUN 可能不够。
- 目前没有断线重连、托管、战绩和聊天。
